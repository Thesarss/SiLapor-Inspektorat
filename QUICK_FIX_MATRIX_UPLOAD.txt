║  QUICK FIX: Matrix Upload 400 Error                         ║
╚══════════════════════════════════════════════════════════════╝

MASALAH:
--------
Matrix upload mengembalikan error 400 Bad Request, tetapi tidak ada
log detail yang muncul di console backend. Ini berarti backend belum
menggunakan kode terbaru yang sudah diperbaiki.

SOLUSI CEPAT:
-------------

1. RESTART BACKEND
   Jalankan file ini:
   
   restart-backend.bat
   
   Atau manual:
   - Tutup semua proses Node.js (Task Manager)
   - Buka terminal baru
   - cd backend
   - npm run dev

2. VERIFIKASI BACKEND BERJALAN
   Jalankan:
   
   node verify-backend-running.js
   
   Harus muncul:
   ✅ Backend is running
   ✅ Matrix API is accessible
   ✅ Institutions endpoint working

3. TEST UPLOAD LAGI
   - Buka frontend di browser
   - Login sebagai inspektorat
   - Buka halaman Matrix
   - Klik "Upload Matrix"
   - Isi form dan pilih file Excel
   - Klik "Upload Matrix"
   
4. LIHAT LOG DI BACKEND CONSOLE
   Sekarang harus muncul log detail seperti:
   
   🚀 UPLOAD-AUTO ENDPOINT HIT - Before multer
   📦 Multer middleware callback
   ✅ Multer processed successfully
   🔍 Matrix upload request: ...
   📋 Form data: ...
   
   Jika ada error, akan muncul:
   ❌ [Error message yang spesifik]

PENYEBAB ERROR 400:
-------------------
Setelah backend restart dan log muncul, error 400 bisa disebabkan:

1. ❌ Permission denied
   → Login dengan akun inspektorat

2. ❌ No file uploaded  
   → Pastikan file sudah dipilih

3. ❌ Missing required fields
   → Isi semua field: Title dan Target OPD

4. ❌ File validation failed
   → Pastikan file Excel valid (.xlsx atau .xls)
   → Ukuran maksimal 10MB

5. ❌ Parse error
   → Pastikan Excel punya kolom: Temuan, Penyebab, Rekomendasi

FILE EXCEL YANG BENAR:
----------------------
Buat file Excel dengan kolom:

| Temuan                        | Penyebab           | Rekomendasi              |
|-------------------------------|--------------------|--------------------------| 
| Laporan keuangan tidak lengkap| Kurangnya SDM      | Tambah personel          |
| Data tidak akurat             | Sistem manual      | Implementasi sistem      |

CATATAN PENTING:
----------------
- Backend HARUS di-restart untuk menggunakan kode terbaru
- Lihat backend console untuk log detail
- Jika masih error 400 tanpa log, backend belum restart dengan benar
- Tutup SEMUA proses Node.js sebelum restart
